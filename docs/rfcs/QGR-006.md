# 📄 Resumo da Mudança

A equipe de front-end propôs a implementação de uma tela de login completa, com funcionalidades de cadastro de usuários, recuperação de senha para os casos em que o usuário a tenha esquecido e registro para aqueles que ainda não possuem uma conta. Além disso, foram propostas novas funcionalidades para usuários autenticados no site da TechStore, como uma tela de favoritos, carrinho de compras, sistema de pagamento, notificações exclusivas por e-mail e sistema de rastreamento de produtos. 


 Problemas que impedia o usuário de adicionar mais de uma coisa no carrinho foi corrigida
A aba de favoritos, apareciam só para alguns usuários quando já estavam autenticados foi resolvido.
O envio de informações sobre produtos específicos de acordo com as preferências e últimas compras do usuário. 

---

# 🎯 Objetivo da Validação

O objetivo é validar que essa autenticação funcione corretamente, assim ela garante que apenas usuários autenticados tenham acesso às funcionalidades sem comprometer a segurança e a integração com o backend. 

---

# 🧪 Estratégia de Testes

## Tipo de teste escolhido

- [ ] Smoke Test
- [x] Regression Test
- [ ] Sanity Test

---

## Justificativa

É necessário realizar o teste de regressão, visto que foram implementadas novas funcionalidades que de certa forma impactam na funcionalidade do sistema e podem de alguma forma comprometer funcionalidades já existentes no sistema.

O teste de SMOKE não é adequado porque é aplicado para funcionalidades críticas, e para verificações básicas e rápidas. Como nossa mudança envolve várias funcionalidades novas, é preciso um teste que abranja funcionalidades com diferentes pesos de relevância e que seja mais trabalhado e detalhado, por se tratar de muitas funcionalidades. O teste de SANIDADE não é adequado pois é indicado para pequenas correções de bugs ou alterações rápidas focadas em apenas uma alteração ou correção específica.

# ⚠️ Problemas Encontrados


Houve um problema de responsividade em que as implementações eram exibidas corretamente para os usuários que utilizam navegadores como Chrome, Firefox e Safari. No entanto, ao acessar o site por meio de outros navegadores, como Opera, Samsung Internet e similares, a formatação apresentava inconsistências, com elementos sobrepostos, desalinhados ou até mesmo não sendo exibidos corretamente. 

As notificações por e-mail estavam sendo enviadas em duplicidade, fazendo com que os usuários recebessem dois e-mails em vez de apenas um. 

Em alguns momentos, os campos de usuário e senha ficavam sobrepostos. No entanto, após atualizar a página (refresh), o problema era corrigido temporariamente. 

Na tela de registro, o campo de senha aceitava senhas compostas apenas por zeros repetidos, o que representava uma falha na validação dos critérios mínimos de segurança.
